# Сделать понятную обработку ошибок открытия repo

## Контекст
Не-git каталог — ожидаемый пользовательский сценарий.

## Цель
Ошибка открытия repo предсказуема и не ломает состояние.

## Подзадачи
1. Преобразовать E_GIT_ERROR в короткий пользовательский текст.
2. Залогировать подробности.
3. Не оставлять UI в полуоткрытом состоянии.
4. Проверить cancel path отдельно от error path.

## Зависимости
- T02_open_repo_action_flow.md

## Артефакты
- изменения в коде
- тесты на заявленный уровень
- обновление документации, если менялся контракт/архитектура

## Acceptance Criteria
- Cancel и invalid-repo различаются по поведению.
- invalid-repo возвращает короткий пользовательский текст без частичного открытия состояния.

## Definition of Done напоминание
- код собран
- тесты проходят
