# Добавить recent repos storage (желательно)

## Контекст
Небольшое улучшение UX, не ломающее MVP, но полезное для следующего запуска.

## Цель
Host помнит недавно открытые repo.

## Подзадачи
1. Сохранить путь после успешного repo.open.
2. Ограничить длину списка.
3. Исключить дубликаты.
4. Подготовить API для будущего recent repos view.

## Зависимости
- T02_open_repo_action_flow.md

## Артефакты
- изменения в коде
- тесты на заявленный уровень
- обновление документации, если менялся контракт/архитектура

## Acceptance Criteria
- Recent repo paths сохраняются между запусками.
- Recent repos не содержат дубликатов и ограничены по длине.

## Definition of Done напоминание
- код собран
- тесты проходят
- документация синхронизирована
