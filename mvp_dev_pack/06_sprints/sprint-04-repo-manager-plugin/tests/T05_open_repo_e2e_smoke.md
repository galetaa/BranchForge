# Покрыть open repo flow e2e-smoke тестом

## Контекст
Нужно зафиксировать первый полностью работающий вертикальный срез.

## Цель
Первый жизнеспособный demo flow защищён тестом.

## Подзадачи
1. Старт host с plugin runtime.
2. Вызов palette action.
3. Эмуляция выбора каталога.
4. Проверка store и events после repo.open.

## Зависимости
- T02_open_repo_action_flow.md
- T03_empty_state_and_open_hint.md

## Артефакты
- изменения в коде
- тесты на заявленный уровень
- обновление документации, если менялся контракт/архитектура

## Acceptance Criteria
- E2E smoke проходит без ручного вмешательства.
- Smoke suite покрывает success и cancel варианты открытия repo.

## Definition of Done напоминание
- код собран
- тесты проходят
- документация синхронизирована
