# Добиться корректной реактивности status panel

## Контекст
UI должен сам обновляться после job без ручного refresh пользователя.

## Цель
Status panel ощущается живой, а не статичной.

## Подзадачи
1. Перестроить model после `event.state.updated(kind=status)`.
2. Обновлять availability кнопок.
3. Обработать случай пустых групп.
4. Проверить состояние после error.

## Зависимости
- T03_stage_selected_action.md
- T04_unstage_selected_action.md

## Артефакты
- изменения в коде
- тесты на заявленный уровень
- обновление документации, если менялся контракт/архитектура

## Acceptance Criteria
- Пользователь видит актуальный статус без перезапуска view.
- После stage/unstage статусные группы обновляются автоматически.

## Definition of Done напоминание

- тесты проходят
- документация синхронизирована
