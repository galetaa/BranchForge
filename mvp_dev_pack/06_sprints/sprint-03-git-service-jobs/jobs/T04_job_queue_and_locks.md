# Сделать job queue и lock policy

## Контекст
Git операции нельзя выполнять хаотично.

## Цель
Host не запускает конфликтующие операции одновременно.

## Подзадачи
1. Добавить job states.
2. Добавить repo-scoped locks.
3. Задать READ/INDEX_WRITE/REFS_WRITE/NETWORK.
4. Сделать последовательное исполнение конфликтующих jobs.

## Зависимости
- T01_git_command_runner.md

## Артефакты
- изменения в коде
- тесты на заявленный уровень
- обновление документации, если менялся контракт/архитектура

## Acceptance Criteria
- Lock behavior покрыт тестом.
- READ jobs могут выполняться параллельно, write jobs блокируются при конфликте.

## Definition of Done напоминание
- код собран
- тесты проходят
- документация синхронизирована
- сценарий задачи воспроизводим
