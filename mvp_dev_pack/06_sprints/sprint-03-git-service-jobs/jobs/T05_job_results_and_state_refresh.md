# Связать job results с обновлением store

## Контекст
После каждого op UI должен видеть актуальное состояние.

## Цель
Job system становится полезным для product flow.

## Подзадачи
1. После repo.open обновлять repo/status/selection.
2. После stage/unstage/commit обновлять нужные snapshots.
3. Публиковать `event.job.finished`.
4. Гарантировать консистентное состояние после ошибки.

## Зависимости
- T03_repo_open_and_status_handlers.md
- T04_job_queue_and_locks.md

## Артефакты
- изменения в коде
- тесты на заявленный уровень
- обновление документации, если менялся контракт/архитектура

## Acceptance Criteria
- После завершения job store и events синхронны.

## Definition of Done напоминание

- тесты проходят
- документация синхронизирована
