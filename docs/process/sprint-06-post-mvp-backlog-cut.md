# Sprint 06 Post-MVP Backlog Cut

## Scope Freeze for MVP v0.1

The following areas are explicitly out of MVP release scope and moved to post-MVP backlog:

- Commit history graph and advanced history navigation.
- Rebase/cherry-pick workflows.
- Merge conflict resolution UI.
- Remote operations UX (push/pull/fetch flow polish).
- Installer/signing/distribution channels beyond local package layout.

## Non-Blocking Rule

All items above are non-blocking for v0.1 release candidate and final MVP close.

## Next-Step Buckets

1. Core Git UX expansion (history/rebase/conflicts).
2. Distribution hardening (installer, signing, auto-update).
3. Extended plugin ecosystem and richer host UI signals.

## Decision

Sprint 06 keeps MVP release criteria focused on:

- open repository
- show status groups
- stage/unstage selection
- create commit from staged changes
