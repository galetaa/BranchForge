# Sprint 06 Packaging and Local Distribution

## Scope

This document closes Sprint 06 T04 by defining the local package layout and verification flow.

## Package Layout

Generated by `scripts/package-local.sh` into `target/tmp/local-package/` (or custom output path):

- `bin/app_host` - host executable
- `plugins/repo_manager` - repo manager plugin binary
- `plugins/status` - status plugin binary
- `plugins/history` - history plugin binary
- `plugins/branches` - branches plugin binary
- `plugins/tags` - tags plugin binary
- `plugins/compare` - compare plugin binary
- `plugins/diagnostics` - diagnostics plugin binary
- `README.txt` - quick usage notes
- `manifest.txt` - build metadata (`commit`, `built_utc`, layout version)

## Build and Package Command

```bash
./scripts/package-local.sh
```

Optional custom output path:

```bash
./scripts/package-local.sh ./target/tmp/my-package
```

## Local Verification

1. Ensure binaries exist in package directories.
2. Confirm `manifest.txt` has commit and UTC timestamp.
3. Run host from package root:

```bash
cd ./target/tmp/local-package
./bin/app_host
```

## Notes

- This layout is for local RC validation and smoke checks.
- Nightly artifacts use the same bundled plugin set as this local package baseline.
- Packaging strategy can be extended later for installer/signing workflows.
