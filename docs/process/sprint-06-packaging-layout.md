# Sprint 06 Packaging and Local Distribution

## Scope

This document closes Sprint 06 T04 by defining the local package layout and verification flow.

## Package Layout

Generated by `run release.package_local` in the runtime host into `target/tmp/local-package/` (or custom output path):

- `bin/app_host` - host executable
- `plugins/repo_manager` - repo manager plugin binary
- `plugins/status` - status plugin binary
- `plugins/history` - history plugin binary
- `plugins/branches` - branches plugin binary
- `plugins/tags` - tags plugin binary
- `plugins/compare` - compare plugin binary
- `plugins/diagnostics` - diagnostics plugin binary
- `README.txt` - quick usage notes
- `manifest.txt` - text build metadata
- `manifest.json` - structured release metadata (`version`, `channel`, `rollback_from`, `platform`)
- `rollback.json` - rollback policy metadata
- `release_notes.md` - generated release notes bundle
- `changelog.md` - packaged changelog snapshot
- `support.md` - packaged support/troubleshooting snapshot
- `sha256sums.txt` - artifact checksum manifest
- `sha256sums.sig` - detached checksum signature
- `sha256sums.pub` - verification key for the checksum signature
- `signing.json` - signing metadata

## Build and Package Command

```bash
cargo run -p app_host -- --command "run release.package_local"
```

Optional custom output path:

```bash
cargo run -p app_host -- --command "run release.package_local ./target/tmp/my-package"
```

Release archive wrapper:

```bash
BRANCHFORGE_RELEASE_CHANNEL=stable cargo run -p app_host -- --command "run release.package"
```

## Local Verification

1. Ensure binaries exist in package directories.
2. Confirm `manifest.txt` and `manifest.json` have commit, UTC timestamp, channel, and platform metadata.
3. Verify signed checksums:

```bash
openssl dgst -sha256 -verify ./sha256sums.pub -signature ./sha256sums.sig ./sha256sums.txt
```

3. Run host from package root:

```bash
cd ./target/tmp/local-package
./bin/app_host
```

## Notes

- This layout is the local RC validation baseline and the input for `run release.package`.
- Nightly artifacts use the same bundled plugin set as this local package baseline.
- Release notes are generated by `run release.notes`.
- Signing is handled by `run release.sign`. If `BRANCHFORGE_SIGNING_KEY` is unset, runtime signing emits an ephemeral dev signature for local verification flows.
